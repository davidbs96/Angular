"Hola Mundo" en Angular

El primer paso siempre que se aprende un nuevo lenguaje de programación es hacer el clásico “Hola Mundo”, y aunque Angular no es un lenguaje de programación como tal, sino un web framework, sí que se basa en el lenguaje Typescript.

Sea como sea, vamos a por el "Hola-Mundo":

Crear el espacio de trabajo

Una vez instalado Angular CLI podemos crear un nuevo espacio de trabajo o aplicación:

ng new "nombre-app" Como vamos a crear un “Hola Mundo” nuestro espacio de trabajo se llamará hello-world:
ng new hello-world

El comando new así como el comando ng tienen muchas variantes, si quieres conocer más acerca de los comandos que puedes realizar deberías visitar la guía de Angular CLI.

Tras introducir el comando, Angular realiza dos preguntas:



Would you like to add Angular routing? (y/N) Pregunta si se desea agregar el módulo de rutas a nuestra aplicación.

Este módulo consiste en un servicio que presenta un determinado componente a partir de la URL suministrada. No te preocupes si no te suenan conceptos como módulo, servicio o componente. Los detallaremos más adelante.

De momento estamos aprendiendo, así que escribimos N (o no escribimos nada) y presionamos ‘Enter’.



Which stylesheet format would you like to use? Angular quiere que indiquemos el formato de nuestras hojas de estilo.

Por simplicidad, utilizaremos CSS.

Tras esto, nuestro proyecto se creará, y podremos ver que Angular ha creado una carpeta llamada “hello-world” con todos los ficheros necesarios para nuestra aplicación.

Ficheros de un proyecto de Angular



  • Carpeta e2e: Esta carpeta, cuyo nombre es "end to end", contiene una serie de ficheros que se encargaran de realizar test automáticos, como si un usuario real interactuara con nuestra app.
  • Carpeta node_modules: Todos los paquetes necesarios instalados, los plugins también se instalarán ahí.
  • Carpeta src: la carpeta donde están los archivos fuentes de nuestro proyecto
  • .editorconfig: Configuración del editor, pensado para proyectos grandes, para que todos los integrantes trabajen con la misma configuración y evitar futuros problemas de incompatibilidades.
  • .gitignore: Archivos a ignorar por Git
  • angular.json: Configuración de Angular. Rutas, nombres, versiones, etc.
