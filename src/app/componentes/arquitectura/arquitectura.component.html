Arquitectura

En esta sección os hablaremos sobre las cuatro tipos de clases principales.



Componentes



Los componentes son clases que controlan cómo se comporta un trozo de la pantalla, llamado vista. Usando el decorador @Component transformaremos dicha clase en un componente. Dentro de este decorador se define un selector, el cual identifica el nombre del componente, una plantilla para nuestro html y un estilo CSS para la misma.
Un componente de Angular puede a su vez contener más componentes. Para ello, usaremos Propiedades (@Input()) para envío de datos y Eventos (@Output()) para recibir datos. Para implementar esta comunicación usaremos Bindings (o uniones). Tenemos tres tipos de Bindings. El primero, el Binding de propiedad, nos servirá para asignar un valor a uno de los Inputs del componente. Cada vez que cambie este valor el componente lo va a recibir. El segundo, el Binding de Evento, nos servirá para que cada vez que el componente hijo emita un evento este lo notificará al componente padre. Y el tercero, el Binding Doble, que es una combinación de los dos anteriores. Cada vez cambie el valor se notificará al hijo y cada vez que cambie el hijo se notificará al padre.

Directivas

Una directiva es una clase especial que define palabras clave que podremos usar dentro de nuestros templates. Una directiva puede ser estructural si modifica el diseño o puede ser de atributo, que modifica la aparencia o comportamiento de un componente.

Módulos



Es la clase principal. Todas las aplicaciones tendrán al menos un módulo principal. En un módulo se especifica qué dependencias tiene la aplicación, qué componentes va a contener y cual va a ser el principal.

Servicios

Son clases que pueden ser usadas por los componentes para pedir datos u operaciones. Deberás usar servicios para aislar toda la lógica de negocio de tu aplicación. Suelen ser usados para hacer peticiones a APIs aunque Angular también ofrece un mecanismo para poder usar un mismo servicio dentro de múltiples componentes llamado Inyección de dependencias.


La inyección de dependencias es un patrón de diseño donde se pasan los objetos por parámetro a una clase en vez de ser la propia clase la que instancie esos objetos. De esta manera la clase inyectada es mucho más fácil de testear ya que no tiene dependencias directas. En Angular solo tendremos que especificar en el módulo principal de tu aplicación qué clases queremos hacer inyectables. De este módulo, cada vez que utilices un constructor, Angular lo hará por ti.
